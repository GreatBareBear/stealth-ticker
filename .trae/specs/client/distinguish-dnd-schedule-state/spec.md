# 区分“开启时间段免打扰”和“关闭”状态 Spec

## Why
当前免打扰入口在“关闭”和“已开启时间段但当前不在生效时段”两种情况下视觉一致，用户难以判断是否已配置时间段免打扰，容易造成误解和遗漏设置。

## What Changes
- 为免打扰入口新增“已开启时间段（未生效）”的独立视觉状态，保证与“完全关闭”可一眼区分，同时保持整体极简风格。
- 在免打扰设置弹窗的“当前状态”区域，同步区分“未开启”和“已开启时间段（未生效）”，减少认知歧义。

## Impact
- Affected specs: 免打扰入口状态展示、免打扰设置弹窗状态展示。
- Affected code: `client/src/renderer/src/components/settings/StocksTab.tsx`

## ADDED Requirements
### Requirement: 时间段已开启状态提示
系统 SHALL 在时间段免打扰已开启但当前未生效时，提供独立且简洁的视觉提示。

#### Scenario: 时间段已开启但未生效
- **WHEN** 用户已开启时间段免打扰（存在有效的开始/结束时间）且当前时间不在免打扰区间
- **THEN** 底部免打扰入口应呈现“已开启时间段（未生效）”的视觉提示
- **AND** 该入口仍可点击打开免打扰设置弹窗

## MODIFIED Requirements
### Requirement: 免打扰入口状态映射
免打扰入口必须区分以下三种状态（从低到高提示强度）：
- 关闭：默认低调显示
- 时间段已开启但未生效：保持低调但可识别为“已配置”
- 生效中（时间段生效或临时暂停）：高亮显示并体现“生效/倒计时”
