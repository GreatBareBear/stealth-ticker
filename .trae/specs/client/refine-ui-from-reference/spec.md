# 参考 UI 设计元素借鉴 Spec

## Why
当前项目的设置界面功能已覆盖，但信息密度与控件组织方式仍偏“表单化”。参考图展示了一套更适合“办公隐蔽工具”的设置体验：更强的分组语义、更轻量的控件形态、更明确的解释文案。

## What Changes
- 梳理参考图中可复用的 UI 结构与控件模式，形成可落地到当前项目设置页的设计建议清单。
- 输出“建议采用 / 可选采用 / 不建议采用”的元素分级，并映射到当前项目的具体落点（Settings / 显示 / 高级 / 数据看板等）。
- **不在本阶段改动代码**；仅产出可执行的 UI 改造规格与实施任务拆解，等待您确认后再进入实现。

## Impact
- Affected specs: 设置页信息架构、控件选型与交互一致性、可读性与隐蔽性（低视觉噪音）。
- Affected code（确认采纳后）: [Settings.tsx](file:///workspace/client/src/renderer/src/pages/Settings.tsx)，settings 组件目录（DisplayTab/AdvancedTab/DashboardTab 等），以及与 store 绑定的 settings schema。

## ADDED Requirements
### Requirement: 参考 UI 元素分析与落地建议
系统 SHALL 提供一份可审阅的“参考 UI 元素借鉴建议”，包含元素说明、价值点、落地点、风险与替代方案，并支持您选择是否采纳。

#### Scenario: 您确认采纳的元素进入实施
- **WHEN** 您对建议清单逐条标注“采纳/不采纳/需要改造后采纳”
- **THEN** 后续实现阶段仅落地被采纳的元素，并以最小改动方式融入现有设置页与持久化配置

## 分析结果与建议（供您确认）

### A. 强烈建议借鉴（高价值 / 低风险）
1. **分组结构：大标题 + 细分区块 + 解释文案**
   - 参考图特征：每个模块（如“股票表格自定义列”“基金表格自定义列”）有明显区块边界；区块下方有一行解释文本。
   - 价值：降低学习成本；在功能多的设置页中“自解释”；适合隐蔽工具的“快速扫读”。
   - 落点建议：
     - Settings 页各 Tab 内采用统一的 Section 结构：Section 标题（加粗）+ 控件行 + 辅助说明。
     - 将目前散落的 Divider/Title 风格统一为同一套间距/字体层级。

2. **“两列对齐”设置行（Label 左、Control 右）**
   - 参考图特征：开关类与选择类控件均保持横向对齐；说明文本紧随其下。
   - 价值：提升可扫描性；减少视觉跳跃；适合高频调整（透明度、主题、右键等）。
   - 落点建议：
     - DisplayTab/AdvancedTab 将所有配置改为“统一行高、固定 label 宽度”的布局；减少多层嵌套表单造成的对齐偏差。

3. **轻量“胶囊/Chip”多选（用于列选择）**
   - 参考图特征：多选项以可点击的小方框 + 标签，排列为多行网格；选中态突出（蓝色描边/填充）。
   - 价值：比表格 + Checkbox 更直观；适合“自定义列”这种多选密集场景。
   - 落点建议：
     - 新增/改造“数据看板”Tab：将“显示哪些列”的配置以 Chip 多选呈现。
     - 可扩展到“股票行情主窗显示字段选择”（例如是否显示涨跌幅/现价/备注等）。

4. **二元/少量选项使用“分段控制（Segmented）”**
   - 参考图特征：如“表格文字大小：大/默认/小”采用三段式选择；“是否开启页面右键：关闭/开启”采用分段按钮。
   - 价值：比下拉更快；更像“快捷开关”；减少点击层级。
   - 落点建议：
     - DisplayTab：将字体大小从下拉改为分段（大/默认/小）或更少档位。
     - 右键菜单开关、主题浅色/深色可统一为分段/开关的固定样式。

5. **滑块配置带“目的性说明”**
   - 参考图特征：透明度/灰度滑块下方有一句“通过调节透明度，可方便上班使用”。
   - 价值：让“隐蔽性”目标显性化；提升产品叙事一致性。
   - 落点建议：
     - AdvancedTab：透明度/背景色/灰度（若采纳）增加简短说明行，避免用户不知道用途。

### B. 建议选择性借鉴（价值中等 / 需匹配产品定位）
1. **“界面灰度”滑块（把界面趋近办公软件）**
   - 价值：更隐蔽（趋近文档/IDE 灰阶）；对“安全盯盘”强相关。
   - 风险：需要确定灰度作用范围（仅主悬浮窗/设置页/全局）；还需处理图表与涨跌颜色的可辨识度。
   - 落点建议：
     - 首版建议只作用于“行情悬浮窗”背景与文本（不影响涨跌色），并提供“0%~100%”控制。

2. **“自定义列影响面板宽度”的提示逻辑**
   - 价值：避免用户打开过多列导致拥挤；对可读性有帮助。
   - 风险：当前悬浮窗采用弹性布局，需定义“列”概念；否则提示容易变成噪音。
   - 落点建议：
     - 仅在“数据看板/表格型页面”启用此提示；对悬浮窗使用“紧凑模式/自动换行”替代。

3. **“快捷键（老板键）设置”入口与解释文案**
   - 价值：提高发现性；减少用户误解（隐藏/退出差异）。
   - 风险：需要与现有全局快捷键实现一致，避免重复配置字段（目前已有 bossKey*）。
   - 落点建议：
     - AdvancedTab 统一为“开启老板键 + 组合键 + 动作（隐藏/退出）+ 说明文案”，并在 UI 上更接近参考图的简洁样式。

### C. 暂不建议直接借鉴（可能引入不一致或技术负担）
1. **“大量独立页面”式设置拆分**
   - 参考图更像“多页面设置中心”，而当前项目是 Tab 结构。
   - 建议：继续保持 Tab，但在 Tab 内采用参考图的 Section/Row 组织方式即可。

2. **过强的品牌化配色/描边**
   - 参考图使用高对比蓝色描边会提升“可见度”，但隐蔽工具应减少视觉噪音。
   - 建议：保留蓝色为强调色，但默认降低饱和度、减少描边面积，更多使用轻量的选中态（浅底色/细描边）。

## MODIFIED Requirements
